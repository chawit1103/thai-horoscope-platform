# High-Risk PR Checklist

Use this checklist for any PR touching auth, privacy, payment, webhooks, notification delivery, audit logs, or astro calculation integrity.

## General

```text
[ ] Scope is limited to this PR
[ ] No .codex/ files committed
[ ] No .env files committed
[ ] No production secrets
[ ] No unrelated UI/refactor changes
[ ] Tests pass
[ ] Codex review has no major issues
```

## Auth / Admin

```text
[ ] Server actions verify auth server-side
[ ] No caller-controlled role/session is trusted
[ ] Cookies are signed/derived where needed
[ ] Audit partitions use trusted server state
[ ] Denied actions do not log attacker-controlled raw input
```

## Privacy / Data

```text
[ ] Export scoped to current user/session only
[ ] Delete removes or suppresses derived artifacts
[ ] Deleted/deactivated users excluded from notification
[ ] PII not logged
[ ] Consent actions audited without sensitive metadata
```

## Email / LINE / Notification

```text
[ ] No real provider calls in tests
[ ] Sandbox/test mode cannot send real messages
[ ] Webhooks verify signatures and fail closed
[ ] Retry/idempotency keys are stable and valid
[ ] Duplicate sends prevented
[ ] Unsubscribe/blocked/bounced states honored at dispatch time
[ ] Fallback does not send after ambiguous provider failures
```

## Payment / Subscription

```text
[ ] Client-side payment status is never trusted
[ ] Webhook signature verified before processing
[ ] Webhooks idempotent before side effects
[ ] Checkout completion bound to stored server-created checkout session
[ ] User/plan entitlements derived from server-side stored checkout/subscription records
[ ] No card data stored
[ ] No raw payment payload logged
[ ] Duplicate receipt hooks prevented
```

## Astro Calculation

```text
[ ] Engine/profile compatibility enforced
[ ] Ephemeris license guard documented and enforced
[ ] No runtime ephemeris downloads
[ ] No large ephemeris binaries committed
[ ] Ephemeris fingerprint includes content/size
[ ] Ayanamsa metadata present
[ ] calculation_hash deterministic and uses canonical inputs
[ ] Unknown birth time does not claim reliable houses/ascendant
[ ] Fractional-second datetime inputs handled consistently
[ ] No interpretation prose generated by calculation engine
```
